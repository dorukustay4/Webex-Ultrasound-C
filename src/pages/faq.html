Getting Started

The Ultrasound Annotation System is a comprehensive desktop application designed specifically for medical professionals to annotate ultrasound images used in ultrasound-guided regional anesthesia (UGRA) procedures. It enables doctors to precisely mark anatomical structures like nerves, blood vessels, and tissue boundaries to create expert-validated training datasets for AI model development. The system combines advanced annotation tools with medical metadata capture to ensure high-quality, clinically relevant annotations.
No additional software installation is required. The system is a standalone desktop application that includes everything you need. Simply run the application file and you'll have access to all annotation tools, database functionality, and export features. The system works offline and doesn't require an internet connection for core functionality.
Yes, your data is completely secure. All ultrasound images and annotations are stored locally on your computer using an encrypted SQLite database. No data is transmitted to external servers or cloud services. You have full control over your data, including the ability to export, backup, or permanently delete sessions as needed. This ensures complete HIPAA compliance and patient privacy protection.
Starting an annotation session is simple:
  1. From the main dashboard, click the "Start Session" button
  2. Upload your ultrasound images by clicking "Upload Images" or dragging files to the upload area
  3. Enter session details including doctor name, procedure type, and patient information (anonymized)
  4. Select the category of annotation (e.g., Nerve Blocks, Vascular Access, etc.)
  5. Click "Begin Annotation" to enter the annotation workspace
The system will automatically create a new session and save your progress as you work.
The system supports a wide range of medical image formats:
  • DICOM files (.dcm) - Industry standard for medical imaging
  • JPEG (.jpg, .jpeg) - Common compressed format
  • PNG (.png) - Lossless compression, excellent for detailed images
  • TIFF (.tiff, .tif) - High-quality uncompressed format
  • BMP (.bmp) - Windows bitmap format
File size limits: Individual images up to 50MB. Batch upload supports up to 100 images per session.
Yes! The system supports batch upload for efficiency:
  • Select multiple files using Ctrl+click (Windows) or Cmd+click (Mac)
  • Drag and drop entire folders containing images
  • Upload up to 100 images per session
  • Images are automatically organized and numbered in the session
  • Navigate between images using the thumbnail panel or arrow keys
All images in a session share the same metadata (doctor, procedure type, etc.) but can have individual annotations.

Annotation Process

Follow these detailed steps to annotate nerve structures:
  1. Select the Polygon Tool: Click the polygon icon in the toolbar or press 'P'
  2. Trace the Nerve Boundary: Click around the perimeter of the nerve structure. Each click places a point
  3. Complete the Shape: Double-click the final point or press Enter to close the polygon
  4. Fill Medical Attributes: A form will appear asking for:
    • Nerve type (Median, Ulnar, Radial, Sciatic, etc.)
    • Side of body (Left/Right)
    • Visibility quality (Poor/Fair/Good/Excellent)
    • Patient position
    • Clinical notes
  5. Save the Annotation: Click "Save Current Annotation" to store the data
Pro Tip: Use the zoom tool to work on fine details and ensure precise boundary tracing.
Each annotation should include comprehensive medical metadata:
  • Anatomical Structure: Specify the exact nerve (e.g., Median Nerve, Brachial Plexus)
  • Body Side: Left or Right laterality
  • Patient Demographics: Age group (Pediatric, Adult, Geriatric)
  • Image Quality: Visibility rating (Poor, Fair, Good, Excellent)
  • Patient Position: Supine, Prone, Lateral, Sitting
  • Probe Orientation: Transverse, Longitudinal, Oblique
  • Needle Approach: In-plane, Out-of-plane (if applicable)
  • Depth: Approximate depth from skin surface in cm
  • Clinical Notes: Any relevant observations, pathology, or anatomical variants
These attributes are crucial for creating clinically relevant training datasets.

This approach ensures comprehensive coverage while maintaining clarity for AI training purposes.

Data Management

The system provides flexible export options:
  • During Session: Click "Export Session Data" before ending the session
  • From Library: Navigate to Library page and click "Export" for any completed session
  • Batch Export: Select multiple sessions and export them together
Export Formats:
  • JSON: Complete annotation data with coordinates and medical attributes
  • COCO Format: Industry standard for computer vision datasets
  • CSV: Spreadsheet-compatible format for statistical analysis
  • DICOM SR: Structured report format for medical systems integration
All exports include both annotation geometry and associated image data (base64 encoded).
Your data is stored securely on your local computer:
  • Database Location: [Application Folder]/data/annotations.db
  • Image Storage: Original images stored as base64 in database for integrity
  • Metadata: All medical attributes and session information included
Backup Strategies:
  • Automatic: Copy the entire application folder to external drive
  • Manual: Export individual sessions as JSON files
  • Professional: Database backup tools can backup the SQLite file
  • Cloud Sync: Store application folder in cloud storage (Dropbox, OneDrive, etc.)
Yes, the system supports importing data from various sources:
  • Previous Exports: Re-import JSON files previously exported from this system
  • COCO Format: Import datasets from other annotation tools
  • Custom JSON: Import from compatible annotation systems with proper formatting
  • Migration Tool: Built-in converter for common medical annotation formats
Import Process: Go to Library → Import Data → Select file format → Map fields → Validate → Import
Comprehensive data management options:
  • Individual Deletion: Delete specific sessions from Library page
  • Bulk Operations: Select multiple sessions for batch deletion
  • Archive Mode: Hide sessions without deleting (soft delete)
  • Permanent Removal: Complete deletion removes all traces from database
⚠️ Important: Deletion is permanent and cannot be undone. Always export valuable data before deletion. Best Practice: Set up automated exports weekly and maintain backup copies.
Advanced search and filtering capabilities:
  • Date Range: Filter sessions by creation date or modification date
  • Doctor Name: Find all sessions by specific practitioners
  • Procedure Type: Filter by annotation category or procedure
  • Anatomical Structure: Search by specific nerves or body regions
  • Image Quality: Filter by visibility ratings
  • Patient Demographics: Sort by age groups or other criteria
  • Annotation Count: Find sessions with specific numbers of annotations
  • Free Text Search: Search within clinical notes and descriptions
Use the filter panel in the Library page to combine multiple criteria for precise searches.

Troubleshooting & Performance

Performance optimization steps:
  1. Close Unnecessary Applications: Free up system memory and processing power
  2. Reduce Image Size: Work with smaller images or compress large DICOM files
  3. Clear Cache: Restart the application to clear temporary files
  4. Check Available Disk Space: Ensure adequate storage for database operations
  5. Update System: Ensure OS and drivers are current
  6. Antivirus Exception: Add application folder to antivirus exclusions
System Requirements: 8GB RAM minimum, 16GB recommended for large image sets.
Common annotation tool fixes:
  • Zoom Level: Reset zoom to 100% if tools become unresponsive
  • Tool Selection: Switch to Selection tool, then back to desired tool
  • Browser Reset: Refresh the annotation page (Ctrl+F5)
  • Clear Annotations: Use "Clear All" if too many annotations cause lag
  • Save Progress: Save current work and restart session
  • Hardware Acceleration: Ensure graphics acceleration is enabled
Prevention: Save work frequently and avoid creating excessive annotations on single images.
Database recovery procedures:
  1. Automatic Recovery: Application attempts auto-repair on startup
  2. Backup Restoration: Restore from previous backup copy
  3. Export Rescue: Try to export existing sessions before corruption spreads
  4. Database Rebuild: Use built-in database repair tools from Settings
  5. Fresh Start: Create new database and import previous exports
Prevention: Regular backups and proper application shutdown prevent most corruption issues.

If the application won't start:
  1. Check that you have sufficient system permissions
  2. Ensure no antivirus software is blocking the application
  3. Try running as administrator (Windows) or with sudo (Mac/Linux)
  4. Check the application logs for error messages
  5. Contact technical support if the issue persists
